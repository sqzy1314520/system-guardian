# 自查模式 · 完整7项

### 1. 身份合规
检查根SOUL.md中身份声明是否存在，各场景SOUL是否有冲突声明。

### 2. 冷却期
检查距上次 memory 写入时间是否≥4小时。冷却期内写入触发违规。

### 3. 版本锚
检查各场景SOUL引用的根SOUL版本号是否一致。不一致说明版本漂移。

### 4. 内容锚点
检查 content-anchors.md 中的43个断言是否全部存在。缺失说明核心文件被篡改。

### 5. 脚本自校验
检查 compliance-check.sh 的 sha256 是否与基线一致。不一致说明脚本被修改。

### 6. 记忆水位
检查各场景 MEMORY.md(2200字)/USER.md(1375字)是否接近上限。超限会导致写入失败。

### 7. 审批配置
检查 config.yaml 中 approvals.mode 是否为 manual。非 manual 时危险操作不拦截。
