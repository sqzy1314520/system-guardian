# 我写了一个 Hermes Agent 系统卫士，一条命令检查 12 个维度

上周在群里看到有人问：「Hermes 跑着跑着突然崩了，日志在哪里看？」

这个问题我踩过太多次了。所以干脆写了一个工具——**System Guardian（系统卫士）**，一条命令，3 秒跑完 12 项检查，你的 Agent 哪里不对一眼就知道。

---

## 为什么需要这个

Hermes Agent 是一个开源的 AI Agent 框架，能力很强，但有个问题：**配置散落在七八个文件里，没人告诉你哪里不对。**

```
~/.hermes/
├── SOUL.md        ← 你是谁
├── config.yaml    ← 怎么连 API
├── .env           ← 密钥
└── profiles/
    ├── work/      ← 工作场景配置
    ├── study/     ← 学习场景配置
    └── ...
```

少配一个 approvals、写错一个路径、超了 4 小时冷却期写了记忆——这些问题不会报错，但系统行为会慢慢变得奇怪。

你以为是 BUG，其实是配置漂移。System Guardian 就是查这个的。

---

## 一条命令，跑完 12 项

安装后对 Agent 说一句「检查一下」，或者直接命令行跑：

```bash
bash compliance-check.sh
```

输出长这样：

```
═══════════════════════════════════════════
  执行纪律巡检 · 2026-06-20 01:09
═══════════════════════════════════════════

── 1. 身份合规 ──
✅ 根SOUL身份正确: 小智

── 3. 版本锚 ──
✅ 版本锚全部一致 (v3.9)

── 11. 外部记忆 ──
✅ Mnemosyne 读写正常（全链路验证）

═══════════════════════════════════════════
  ✅ 33  ⚠️ 5  ❌ 1
═══════════════════════════════════════════
```

它检查的东西：

| 维度 | 查什么 | 如果不对会怎样 |
|:-----|:-------|:-------------|
| 身份合规 | 各场景身份声明是否冲突 | 场景切换时人格分裂 |
| 版本锚 | 所有 SOUL.md 版本号一致 | 改了根配置忘了同步 |
| 记忆水位 | USER.md/MEMORY.md 字符数 | 超限后新记忆写不进去 |
| cron 调度 | 定时任务是否在跑 | 日报不推送、备份不执行 |
| 心跳 | 上次自检是否在 27h 内 | 系统死了没人知道 |
| 外部记忆 | Mnemosyne 数据库活性 | 对话记忆丢失 |

## 怎么用

两种方式：

**方法一：Hermes Skill 安装**

```bash
hermes skills install https://github.com/sqzy1314520/system-guardian/raw/main/SKILL.md
```

安装后对 Agent 说：
- 「检查一下」→ 10 维健康检查
- 「查查哪里不对」→ 深度诊断

**方法二：命令行直接跑**

```bash
curl -O https://raw.githubusercontent.com/sqzy1314520/system-guardian/main/scripts/compliance-check.sh
chmod +x compliance-check.sh
HERMES_HOME=~/.hermes bash compliance-check.sh
```

不依赖 Hermes，任何 Linux/Mac/WSL 都能跑。

---

## 还带了三个小工具

拆成四个独立组件，按需加载：

- **cron-manager** — 定时任务创建向导，三句话建好 cron job
- **goal-tracker** — 系统目标管理，设定连续达标自动追踪
- **system-onboarding** — 首次安装 Hermes 的开箱引导

安装一个 skill 就够了，其他三个自动可用。
