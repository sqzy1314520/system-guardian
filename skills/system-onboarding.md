# System Onboarding · 系统开箱引导

## 定位

首次安装 Hermes 后的一次性引导工具。完成后再不需要加载此 skill。

---

## 裸机检测（前置判断）

加载后先做三个检查：

```
检查 1: SOUL.md 是否存在？
  → 无 → "你的系统还没建，需要先走构建模式。要帮你建吗？" → 跳转构建
  → 有 → 继续
检查 2: audit 目录是否存在？
  → 无 → "检测到你已有基础配置但未初始化自检系统，要一键补全吗？"
  → 有 → 继续
检查 3: compliance-hash.txt 是否存在？
  → 无 → "首次运行，正在初始化自检基线……"
  → 有 → 继续
```

三项全通过 → 系统已就绪，无需构建。

---

## 构建模式

### 固化内容（自动写入，无需选择）

| 模块 | 说明 |
|------|------|
| 控制台 | control profile + 定位声明（"总管家"） |
| 全局认知声明 | 4条固定模板（你是谁/你和勇哥的关系/你和系统的关系/违规追责） |
| 行为底线 | 4条（被纠即升级/用户视角优先/系统化迁移/记忆写入限频） |
| 6条红线 | 不可逾越的边界（发布确认/删除确认/系统目录写确认等） |
| 内容锚定 | content-anchors.md + compliance-check.sh |
| 开机自检 | .bashrc触发 + control启动指令 |
| 审批配置 | approvals.mode: manual |

### 用户配置对话（最少2-3个问题，一次只问一个）

问题①：怎么称呼你？（用户输入）

问题②：主要用来做什么？
> [1] 工作  [2] 自媒体  [3] 学习  [4] 都有  [5] 其他

问题③：动态选项（根据第②步排除已选，改为"加"其他项）

确认后 → 自动搭建 → "全部完成！试试说'帮我写一段自我介绍'。"

### 完成后引导

建完后立刻给一个低门槛的尝试指令：
> "试试说——帮我写一段自我介绍，看看系统怎么写东西。"
